## Root cause

Inductor's reduction scheduler does not recognize that the layout-changing clone (cat -> reshape -> permute -> clone -> reshape) and the sibling sum.dim_IntList over the same data can share the same tile schedule. The oracle writes layout tiles and accumulates partial column sums in a single pass per source tensor (q/k/v), then finishes with a small final-sum kernel. Inductor emits the layout materialization and the reduction as separate schedules with a full reread of the 25344x2304 matrix.

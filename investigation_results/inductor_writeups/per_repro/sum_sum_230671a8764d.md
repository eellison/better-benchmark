## Pattern Description

Qwen3 RoPE-backward epilogue fused with RMSNorm backward. The graph contains:
- Rotary position embedding backward (slice_scatter, permute, cat of half-rotations)
- RMSNorm backward (pow, mean, rsqrt, mul with weight)
- Two outputs: a [128] weight gradient (column sum) and a [2048, 2048] input gradient
  (transposed layout)

## Root Cause

Inductor lowers the slice_scatter/permute/RMSNorm-backward graph as separate generic
pointwise and reduction work. It materializes the RoPE producer and the RMSNorm gradient
before the sibling reductions. Specifically:
1. The RoPE backward producer is computed and written to memory
2. It is re-read for the RMSNorm row-dot-product reduction
3. It is re-read again for the weight gradient column reduction
4. The transposed output requires a layout-changing store

The oracle computes the RoPE product once per row, reuses it for both the RMSNorm hidden-
dimension dot product AND the weight-gradient accumulation, then writes the transposed
input-gradient side output directly.

## Oracle Strategy

Pure PyTorch structured oracle (no handwritten Triton):
1. Compute rotary_grad = full_3 + rotary_term + getitem_81 * unsqueeze_7
2. Permute to row form
3. Compute weight_grad = sum over (batch, seq, heads) of (row_grad * xhat)
4. Compute input_grad via RMSNorm backward formula
5. Return weight_grad and transposed input_grad

The key insight is that steps 3 and 4 share the same producer (row_grad) and can be
fused into a single pass if the scheduler recognizes the shared dependency.

## Inductor Fix Required

**SCHEDULER_FUSION**: Add a fused multi-output RMSNorm-backward/rotary template that:
1. Keeps the per-row dot product in registers
2. Stores the transposed gradient directly (layout-changing side output)
3. Emits coordinated partial reductions for the weight gradient

The scheduler currently cannot represent a row-persistent reduction whose producer also
has a materialized layout-changing side output AND a cross-row sibling reduction over
only the rotary head dimension.

## Generalizability

MEDIUM-HIGH - This pattern (RoPE + RMSNorm backward with shared producer) appears in
all rotary-embedding models that use RMSNorm (LLaMA, Qwen, Mistral families). The
specific fusion requires recognizing that slice_scatter/permute for RoPE backward and
the RMSNorm row-reduction share a common producer.

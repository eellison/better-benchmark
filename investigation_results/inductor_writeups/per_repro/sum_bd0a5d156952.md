## Full-Scope Contract

The oracle consumes the same three contiguous `float32[128, 128, 56, 56]`
inputs as `repro.py` and returns the same single contiguous `float32[128]`
output with stride `(1,)`. It computes the complete repro expression:

```text
out[c] = sum_{n,h,w} (x[n,c,h,w] + y[n,c,h,w])
                     * sigmoid(a[n,c,h,w])
                     * (a[n,c,h,w] * (1 - sigmoid(a[n,c,h,w])) + 1)
```

It is not a reduction-subset benchmark; the timed path includes the full
sigmoid-derived pointwise producer and the channel reduction.

## Gap Diagnosis

The oracle differs from Inductor by using a hand-written Triton split reduction
that reads each original input once, computes the sigmoid-derived pointwise
producer inside the reduction tile, stores per-channel partials, and finalizes
directly into the returned vector. Inductor already emits the same full-scope
shape of lowering for this graph: a fused pointwise reduction kernel followed
by a tiny finalizer.

Inductor cannot unlock a distinct missing-pattern win here because there is no
multi-output producer sharing, scatter-reduce, algebraic elimination,
cooperative split-K opportunity beyond ordinary partial reductions, or
dependent epilogue omitted by the scheduler. The relevant Inductor change would
be lower-level tuning of the existing pointwise-reduction template for this
channel-reduction layout. The historical-best gate shows that this repro has no
separate actionable floor from the current oracle evidence.

## Oracle Description

Triton one-pass kernel launched with C=640 programs. Each program processes all
N*H*W=8192 spatial positions for one channel, computing:
1. Average-pool backward (expand pooled gradient / HW)
2. SiLU backward (sigmoid * (x*(1-sigmoid)+1))
3. BN backward reduction (grad_sum and centered_grad_sum)
4. BN backward output (subtract mean/variance correction terms, scale by invstd*weight)
5. Weight gradient (centered_grad_sum * invstd)

All in a single pass without materializing the full as_strided_scatter/expand
pool-backward tensor. The oracle reads activation and pooled-grad once, writes
the full output tensor and 640-element weight gradient.

## Diagnosis

Classification: SCATTER_REDUCE

Inductor materializes the zero-fill/as_strided_scatter/expand/div chain as a
full f32[128, 640, 8, 8] intermediate for the average-pool backward, then
fuses the SiLU backward and part of BN backward as pointwise consumers. The
two sum.dim_IntList reductions ([0, 2, 3] dims) each re-read the full tensor.
The oracle eliminates the intermediate by computing the broadcasted pool grad
inline (just mm[n, c] / 64) and fuses both reductions into the same pass.

The 2.15x gap is significant and actionable. The key missed optimization is
that Inductor does not recognize the as_strided_scatter -> expand -> div pattern
as a structured broadcast that can be folded into consumers without materialization.

## Inductor Closure Path

- Implementation track: Teach Inductor's scheduler that as_strided_scatter
  followed by expand/broadcast is equivalent to a simple indexed broadcast
  (no materialization needed), then fuse the reduction consumers.
- Alternative: SCATTER_REDUCE template for average-pool-backward + BN-backward
  that emits a per-channel persistent kernel reading spatial positions once.
- Gating: N*HW must fit in a single Triton program's register/shared memory
  budget (here 8192 elements = 32KB in f32, feasible).
- Benchmark policy: compare persistent one-pass vs Inductor's multi-kernel
  schedule on various N/C/HW combinations.

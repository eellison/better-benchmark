## Diagnosis

The oracle computes the complete M2M100 scaled token embedding, computed-position embedding gather, population var_mean LayerNorm (eps=1e-5), affine epilogue, and three aliasing [8192,1024] view returns in one Triton row kernel. The key optimization is fusing the embedding table gathers (token_table[token_id] and position_table[position_id]) directly into the LayerNorm reduction loop.

## Root cause

The oracle fuses embedding gathers (indexed loads with scalar row arithmetic) into the row-reduction loop, avoiding one intermediate buffer write+read. Inductor's norm canonicalization does not sink row-local embedding gathers through the add into the fixed-hidden reduction loop.

## Recommendation

The gap is marginal (1.06x). The fix would require the scheduler to recognize that row-local gather producers with [rows, hidden] iteration space can be inlined into the LayerNorm reduction. This is the same EMBEDDING_GATHER_LAYERNORM_FUSION pattern seen in other M2M100 repros. Low priority given the tiny gap.

## Diagnosis

The oracle computes the complete MobileBERT softmax-backward return from Repro.forward in one last-dimension row kernel, folding the BMM/dropout mask scaling, broadcast attention mask, natural-exp probability reconstruction, row dot-product reduction, row-mask zeroing, exact final fma epilogue, and final contiguous [1024,128,128] view. Inductor currently lowers the decomposed view/where/add/exp/div/where/mul/sum/fma graph as separate generic producers and reduction/epilogue work over materialized [256,4,128,128] values.

## Root cause

The softmax backward pattern is:
1. Reconstruct probs: `exp(logits + mask_bias - row_shift) / row_denom`
2. Apply zero-row mask
3. Scale BMM gradient by dropout mask
4. Compute row dot product: `sum(scaled_bmm * probs, dim=-1)`
5. Final FMA epilogue: `fma(-probs, row_sum, scaled_bmm * probs)`

Inductor's scheduler fusion does not sink the softmax-backward row reduction through the probability-reconstruction producer and its epilogue consumer while preserving the mask broadcasts and fma boundary. The oracle keeps all values live in registers for a single K_LEN=128 row, avoiding any intermediate materialization.

`repro.py` and returns both original outputs: a contiguous `float32[512]`

## Gap Diagnosis

The oracle differs from Inductor by fusing the per-row channel reduction directly
into the dropout-masked transposed epilogue while computing the orthogonal
per-channel reduction with coalesced row tiles and a small finalize kernel.
Inductor cannot turn this into one ideal multi-output reduction template today
because the two sibling sums reduce different axes and one sum feeds a full
pointwise/layout epilogue, so the scheduler keeps the cross-axis reductions and
transpose-shaped epilogue as ordinary nodes instead of a single fused plan. The
class is `SCHEDULER_FUSION`, but the measured result confirms the current tuned
Inductor path is already near or ahead of this full-scope hand-written artifact.

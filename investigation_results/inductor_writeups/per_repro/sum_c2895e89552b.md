## Graph structure (exact)

```
bmm [384,768,64] -> view/squeeze -> src [96,4,768,64]            # a=96 (=B8*H12), b=4 windows, c=768, k=64
iota(9437184) -> as_strided([96,4,768,64], [98304,16384,64,1], 0) -> clone -> view(flat)
index_put(full(0,[9437184]), [idx_flat], src_flat, accumulate=True)
  -> as_strided([96,1536,64], [98304,64,1], 0)                   # reshape-equivalent (96*98304 == 9437184)
  -> constant_pad_nd([0,0,-256,-256])                            # crop t in [256,1280)
  -> view [8,12,1024,64] -> permute x2 -> clone [1024,8,12,64] -> view_default_4 [1024,8,768]
view_default_4 users:
  (1) sum.dim_IntList([0,1], keepdim=True) -> view [768]          # OUTPUT 1 (full reduction)
  (2) view [8192,768] -> permute -> [768,8192]                    # OUTPUT 2 (DENSE aliased output)
```

Index decode: idx[a,b,c,k] = 98304a + 16384b + 64c + k = 98304a + 64*(256b + c) + k,
i.e. dest = out[a, 256b + c, k] in [96,1536,64]. Overlapping windows: window length
W=768 rows, shift S=256 rows, s_D=4 windows -> each output row receives up to 4
(=ceil(W/S)) source contributions. Duplicate destination indices, hence accumulate.

## Does the landed gather-reduce identity (5489b8c2bb9) apply?

NO. The identity requires ALL consumers of the scatter result to be full reductions
over the scattered dims. Here consumer (2) is a dense [768,8192] transposed output
returned from the graph — the [1024,8,768] clone buffer MUST materialize. No guard
extension in scatter_add_gather_reduce_pass can make the reduction identity legal.

## Correct fix: structured-scatter DECODE (consumer-agnostic, semantics-preserving)

The index is affine (as_strided over iota), so the scatter is exactly an
overlap-add. Identity (verified numerically, max abs diff 0.0 on CUDA):

    index_put(full(0), [affine_idx], src, True).view(96,1536,64)
        == sum_{b=0}^{3} constant_pad_nd(src[:,b], [0,0, 256*b, 768-256*b])

The pad-sum form is pure pointwise (masked loads), so Inductor fuses it through
the crop/permute/clone chain into ONE kernel that writes the dense output and
feeds the reduction directly from bmm — no zero-memset of the 37MB scratch, no
75MB atomic scatter, no re-read.

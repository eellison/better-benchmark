# sum_sum_sum_80113e346555


## Measured Timings
- Oracle: 8.86 us
- Compile (CDT): 13.79 us
- Ratio: 1.56x

- **Gap**: 1.55x (confirmed with CUDAGraph measurement)
- **Classification**: SCHEDULER_FUSION
- **Root cause**: BatchNorm backward: K0 reads 4D inputs for spatial reduction, K3 re-reads the same inputs for the output epilogue. Oracle launches 2048 programs that each reduce a tile in registers then write the output in one pass. Inductor tiles by (batch, channel) = 4096, which prevents epilogue fusion with the reduction.
- **Fix approach**: Scheduler needs a "reduction-with-pointwise-epilogue" enhancement that fuses the epilogue store into the reduction kernel. Note: `linear_reduction_elimination` already fires (4 eliminations) but does not help further because the epilogue is not a simple linear function of the reduction result.

- **Affected repros**: ~10 BN-backward variants

## Diagnosis

The oracle covers the full VGG16 repro scope by computing the low-memory
max-pool-backward scatter, mask overwrite, and returned `[512]` channel sum
directly in source space. It accumulates `full` for masked destination pixels
and adds each `[128,512,7,7]` pool-gradient source only when the corresponding
max-pool destination is not masked, so it never materializes the dense
`[65536,196]` scatter buffer. Inductor cannot do this today because the
scheduler sees `_low_memory_max_pool_offsets_to_indices -> scatter_add -> view
-> where -> sum` as generic tensor work rather than one structured
max-pool-backward gather/reduce. The actionable fix is `SCATTER_REDUCE`: add a
max-pool-backward lowering that derives reductions directly from source
gradients and destination masks.

## Root Cause

This is a GPT-J shifted-label cross-entropy backward pattern where the repro computes:
1. A [128, 50400] gradient tensor via one-hot masked reduction + exp epilogue + residual add
2. Two outputs: `permute_default` (transposed [50400, 128] view of the gradient) and `view_default_5` (column sum [50400])

The codegen appears to incorrectly handle the case where one output is a transposed view of a buffer that should have been written by a reduction kernel. The buffer (`buf1`) that backs the [128, 50400] computation is never allocated -- the second kernel writes only `buf2` (the column sum) and seems to expect `buf1` to exist as an intermediate.

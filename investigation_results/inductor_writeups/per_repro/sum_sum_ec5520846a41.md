# sum_sum_ec5520846a41

## Summary

- Model: timm_tf_efficientnet_b0_train (EfficientNet adaptive-pool backward + SiLU + BN backward)
- Oracle: `oracle_structured_pool_upsample_backward_reduce.py`
- Classification: SCATTER_REDUCE
- Ratio: 1.108x (oracle 46.0us, compile 50.9us)
- Kernel count: Inductor 1 kernel (combo), Oracle 3 kernels (partial sums + finalize + input grad)

## Root Cause

The repro computes the EfficientNet adaptive-average-pool backward feeding a SiLU-gated batch-norm backward. The input is `mm [128, 1280]` (pooled gradient) that gets reshaped/expanded to [128, 1280, 7, 7] via as_strided_scatter, then multiplied by the SiLU derivative of the conv output, followed by channel reductions and dependent full-tensor BN-backward gradient.

The oracle uses a three-phase approach:
1. Partial channel sums (cooperative across N*H*W tiles)
2. Finalize sums and compute scale-gradient output
3. Full input-gradient epilogue using finalized sums

Inductor emits 1 combo kernel that handles this reasonably well (only 1.108x gap), likely due to the combo_kernels config fusing the reduction and epilogue. The remaining gap comes from the structured scatter pattern where the pool gradient `[128, 1280]` is broadcast to `[128, 1280, 7, 7]` -- Inductor materializes this expansion rather than recognizing the structured average-pool-backward source.

## Fix Assessment

# mean_7874ce26c7f3 - MobileNet BN Hardswish Spatial Mean

## Benchmark Results
- Oracle: 21.79 us
- Compiled: 24.35 us
- Ratio: 1.117x (oracle wins)

## Classification
SCHEDULER_FUSION - BN-affine + hardswish + spatial mean arithmetic overhead

## Root Cause

The oracle computes the complete MobileNetV3 BN-inference affine, hard-swish activation (x * clamp(x+3, 0, 6) / 6), fp16 cast, and 7x7 spatial mean in one Triton reduction kernel tiled by (batch, channel). Input is f16[256, 960, 7, 7].

Inductor fuses the same operations into a single kernel but uses less efficient arithmetic:
1. `sqrt` + `reciprocal` instead of `rsqrt` for BN variance normalization
2. The hardswish decomposition (add+clamp_min+clamp_max+mul+div) matches the oracle, but combined with the BN overhead, the total instruction count per element is higher
3. 256*960 = 245,760 rows each reducing 49 elements

The ratio (1.117x) is moderate because the hardswish itself dominates compute and both implementations use the same decomposition. The gap is primarily the rsqrt vs sqrt+reciprocal difference.

## Kernel Count
- Oracle: 1 kernel (BN-affine + hardswish + fp16 cast + spatial mean)
- Inductor: 1 kernel (same fusion, different BN arithmetic)

## Config Exploration
- `combo_kernels = True`: no effect (already one kernel)
- `coordinate_descent_tuning = True`: helps block size tuning but not arithmetic
- The 1.117x gap is small enough that rsqrt canonicalization alone likely closes it

## Why Inductor Cannot Do This Today

Same root cause as other BN-inference repros: the algebraic simplifier does not rewrite `reciprocal(sqrt(var+eps))` to `rsqrt(var+eps)`. The hardswish computation is identical between oracle and Inductor (both use clamp-based decomposition), so the gap is almost entirely from the BN normalization arithmetic.

**Enhancement needed**: Canonicalize `reciprocal(sqrt(x))` -> `rsqrt(x)` in post-grad algebraic rewrites. This single fix would likely close the gap for this repro and many similar BN-inference patterns.

**Affected repro count**: Part of the broader BN-inference rsqrt canonicalization family. This specific MobileNetV3 pattern (BN + hardswish + spatial mean) likely has 2-3 similar repros in the corpus with different shapes.

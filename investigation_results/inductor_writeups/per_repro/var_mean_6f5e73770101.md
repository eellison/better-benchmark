## Root cause: The oracle computes the complete DistilBERT embedding LayerNorm training scope (word embedding gather, position embedding slice/gather, add, fp32 var_mean, affine, inductor-style dropout with p=0.1) in one fused Triton row kernel. Inductor already produces a well-fused kernel for this pattern. The 8% gap is within measurement noise for a bandwidth-dominated workload involving 30522x768 embedding table lookups.

The workload is dominated by:
- 256*128 = 32768 gathers from a 30522x768 f32 embedding table (~90MB read with random access)
- 1*128 gathers from a 512x768 f32 position embedding table
- Addition, row reduction, affine, RNG, dropout, and output store

The random-access embedding gathers dominate runtime, and both Inductor and the oracle must perform the same memory traffic.

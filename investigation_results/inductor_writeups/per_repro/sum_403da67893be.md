## Diagnosis

The oracle computes the complete Qwen cross-entropy-backward dense gradient epilogue in one Triton row-loop kernel. It replaces the materialized one-hot tensor (iota == label) and its full vocabulary-width row sum with the equivalent guarded label scalar formula. The kernel computes shifted exp values, one-hot-scaled correction, bf16 rounding, residual add, and transposed output store per row.

## Root cause

The oracle proves that `sum(one_hot(labels, vocab) * values, dim=-1)` is equivalent to `values[label]` (a scalar gather). Inductor does not perform this algebraic simplification, so it materializes the full [rows, 151936] one-hot mask and reduces across it. This is O(vocab) work per row that the oracle reduces to O(1).

## Recommendation

Requires an algebraic elimination pass in `torch/_inductor/fx_passes/` that recognizes `sum(where(iota == scalar_index, value, 0), reduction_dim)` patterns and rewrites them to a guarded scalar gather. This is the ONE_HOT_REDUCTION_ELIMINATION pattern.

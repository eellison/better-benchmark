## Full-Scope Computation

The repro takes two contiguous `f32[128, 16, 112, 112]` inputs and returns one
contiguous `f32[16]` output:

```python
sig = 1 / (exp(-arg142_1) + 1)
out = (getitem_237 * sig * (arg142_1 * (1 - sig) + 1)).sum(dim=(0, 2, 3))
```

The diagnostic oracle covers that entire scope with the same inputs, output
shape, dtype, and stride. It uses Triton for the timed path; torch is used only
for reference checking and compile baselines.

## Gap Diagnosis

The oracle keeps the sigmoid-backward-like pointwise expression inside the
reduction tile and writes only the final channel result. Inductor already fuses
the surrounding pointwise expression into its generated reduction and emits a
partial-reduction kernel plus a tiny finalize kernel, so this repro is not a
missing multi-output fusion floor. The remaining cost is dominated by two
102.8 MB fp32 input reads and one exp/sigmoid-style operation per element.

No generic scheduler fusion is expected to produce a lower true floor here. A
future Inductor improvement would need a narrowly gated sigmoid/reduction
template that beats the existing tuned historical timing on this exact shape.

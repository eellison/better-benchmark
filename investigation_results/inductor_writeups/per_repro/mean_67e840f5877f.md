# mean_67e840f5877f - BN SiLU Spatial Mean (MobileViT)

## Benchmark Results
- Oracle: 11.26 us
- Compiled: 13.31 us
- Ratio: 1.182x (oracle wins)

## Classification
SCHEDULER_FUSION - BN-affine + SiLU + spatial mean not optimally tiled

## Root Cause

The oracle computes the full MobileViT batch normalization inference (affine transform using per-channel mean/var/weight/bias), SiLU activation, and 8x8 spatial mean into the final contiguous f32 [128, 640] output in one batch/channel-tiled Triton reduction. It reuses per-channel BN parameters (loaded once per channel tile) across the 64-element spatial reduction.

Inductor generates a fused kernel that handles the same operations but:
1. Uses a generic channels-last stride schedule rather than a specialized batch*channel row tiling
2. Repeats per-channel parameter loads/computations (sqrt+reciprocal for variance) more than necessary due to how the reduction loop is structured
3. The oracle uses rsqrt directly while Inductor uses sqrt+reciprocal (algebraic inefficiency)

The spatial reduction (8x8=64 elements) is small enough that the overhead of BN parameter recomputation per element is measurable. The oracle tiles by (batch, channel_block) and reduces all 64 spatial elements in one pass per channel.

## Kernel Count
- Oracle: 1 kernel (BN-affine + SiLU + spatial mean, batch*channel-tiled)
- Inductor: 1 kernel (same fusion, different tiling/arithmetic)

## Config Exploration
- `combo_kernels = True`: no effect (already fused)
- `coordinate_descent_tuning = True`: may tune block sizes but cannot fix the algebraic issue
- The gap is primarily from rsqrt vs sqrt+reciprocal and tiling strategy

## Why Inductor Cannot Do This Today

Two issues:
1. **Algebraic**: Inductor's lowering decomposes `1/sqrt(var+eps)` into `sqrt` then `reciprocal` (2 transcendental ops) instead of using `rsqrt` (1 op). Over 128*640=81920 rows each computing this, it adds up.
2. **Tiling**: The oracle loads per-channel parameters once per (batch, channel_block) tile and reuses them across the 64-element spatial reduction. Inductor's generic reduction schedule may reload or recompute these more frequently.

The channels-last input stride (40960, 1, 5120, 640) means spatial elements are not contiguous in memory - the oracle handles this with explicit indexing while Inductor's generic schedule may not tile optimally.

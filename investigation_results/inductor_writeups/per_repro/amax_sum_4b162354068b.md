## Scope

The oracle covers the full `Repro.forward` computation:

- slice the `[1, 1, 2048, 2048]` bool mask to `[1, 1, 128, 128]`
- view `bmm_46` from `[512, 128, 128]` to `[32, 16, 128, 128]`
- apply the mask with the captured fp32-min fill value
- add the `[32, 1, 128, 128]` fp32 bias
- compute the stable softmax over the last dimension
- return the final contiguous fp32 `[512, 128, 128]` view with stride `(16384, 128, 1)`

## Gap Diagnosis

The oracle uses a single shape-specialized Triton attention-softmax kernel. It
tiles all 16 heads for each batch/query row, reuses the sliced mask and
batch/query bias vector across that head tile, predicates score loads by the
mask, and writes the final view layout directly.

Inductor can fuse the graph, but the generic online-softmax schedule used here
does not express this small-K masked additive-bias attention-specific reuse of
mask and bias across heads. It therefore keeps more generic per-row producer
work around the reduction.

The Inductor fix is a NEW_PATTERN lowering: add a K=128 masked additive-bias
attention softmax template, or extend online-softmax scheduling so broadcast
mask/bias producers can be hoisted and reused across head tiles before
normalization and final view emission.

# sum_12b455c3b1b0

## Compile: 28.0us, Oracle: 15.1us, Gap: 1.85x

## Diagnosis: SCATTER_REDUCE

## Root cause: Inductor emits 6 separate kernels for the Longformer layout assembly: (1) iota, (2) full zeros, (3) fused index_put/scatter, (4-5) two-stage sum reduction, (6) output transpose/permute. The oracle computes the entire scope in 2 kernels by recognizing that the iota/as_strided/index_put pattern produces overlapping writes from 4 source blocks (offsets 0/256/512/768) into the same output positions, and directly reads from the overlapping source positions, accumulates the sum as a byproduct, and writes the final transposed layout -- eliminating the 2.3M-element dense intermediate buffer and 3 extra kernel launches.

## Fix path: Add a pattern-matching pass that recognizes iota/as_strided/index_put with fixed overlap structure, and lowers it as a direct multi-source gather with compatible reduction epilogues. This is a Longformer-specific optimization that would benefit hf_Longformer training specifically.

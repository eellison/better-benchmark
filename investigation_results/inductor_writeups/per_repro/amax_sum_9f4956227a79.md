## Details

- Model: timm_swin_base_patch4_window7_224 (infer+train, 4 shapes)
- Pattern: amax+sum reduction (attention softmax with relative position bias)
- Ops: view x4, index, permute, clone, unsqueeze, add, amax, sub, exp, sum, div, expand
- Shape: [4096,49,49] -> [128,32,49,49] view, [169,32] bias gather via [49,49] index -> softmax -> [4096,49,49]
- Nearly identical to amax_sum_17ab35828f89 (uses view instead of reshape); same fix applies.

## Mechanism

The [32, 49, 49] clone node (result of index+view+permute+clone) had 128 users
(broadcast into 128 windows via unsqueeze+add). `should_realize_on_reuse(128)`
returned False because `num_reads() = 2 <= realize_reads_threshold = 4`.

The fix adds indirect_indexing to the realize-on-reuse heuristic (like transcendentals).
This causes the [32, 49, 49] = 76832-element bias (300KB) to be materialized in a
tiny pointwise kernel, then the softmax reduction reads it directly (coalesced) instead
of replaying the random-access gather 128 times.

Generated kernel structure after fix:
1. Kernel 0 (pointwise, 76832 elements): materializes bias via indirect gather
2. Kernel 1 (persistent reduction, 200704 rows x 49 cols): softmax with direct bias read

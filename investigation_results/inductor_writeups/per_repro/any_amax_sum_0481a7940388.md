# any_amax_sum_0481a7940388

## Classification: MASKED_SOFTMAX_PATTERN

## Pattern

ALBERT masked attention softmax: [512, 512, 512] scores viewed as [8, 64, 512, 512],
stride-zero broadcast bool mask -> 0/-inf bias, any(eq(-inf)) all-masked-row guard,
stable softmax (amax, sub, exp, sum, div), zero fill for all-masked rows, expand,
view back to [512, 512, 512].

## Kernel Count

- Oracle: 1 kernel (fused row-softmax with mask handling)
- Inductor: 1 kernel (persistent fused but with tautological mask overhead)

## Root Cause

Inductor generates a single fused kernel but does not eliminate the mask construction
overhead. The oracle recognizes the stride-zero broadcast mask pattern and efficiently
handles it within the row softmax. The gap (1.11x) is minor and comes from:
1. Extra instructions for mask materialization within the kernel
2. The any() reduction for all-masked-row detection adds overhead

This is the same family as any_amax_sum_97b43144672a (1.10x) and other
masked-softmax repros. The gap is at the margin of measurement noise for this
shape (512x512 rows, large enough to be bandwidth-bound).

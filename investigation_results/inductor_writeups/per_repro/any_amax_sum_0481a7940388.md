## Pattern

ALBERT masked attention softmax: [512, 512, 512] scores viewed as [8, 64, 512, 512],
stride-zero broadcast bool mask -> 0/-inf bias, any(eq(-inf)) all-masked-row guard,
stable softmax (amax, sub, exp, sum, div), zero fill for all-masked rows, expand,

## Root Cause

The any-elimination pass fires (eliminates the any() reduction) but provides
no measurable improvement for this shape because K=512 makes the kernel
bandwidth-bound. The warp-shuffle cost of one extra reduction is negligible
relative to 512 float loads and 512 float stores per row.

The remaining gap comes from:
1. Double-exp computation (prepare_softmax_twopass_fallback)
2. Oracle's conditional row-skip for fully-masked rows
3. Oracle uses tl.exp2 (native PTX) vs libdevice.exp

## Fix Applied

**masked_softmax_any_elimination** pass fires on this repro (eliminates the any()
reduction) but has no measurable perf impact at K=512. The pass is still
semantically correct and reduces instruction count.

File: `/tmp/pytorch-work/torch/_inductor/fx_passes/masked_softmax_any_elimination.py`
Config: `torch._inductor.config.masked_softmax_any_elimination = True` (default)

## Remaining Gap Analysis

At K=512, the gap is dominated by:
1. **Double-exp**: exp computed in reduction for sum_exp, then again in epilogue
2. **Conditional row-skip**: oracle skips entire 512-element row for masked rows
3. **exp2 vs libdevice.exp**: native PTX vs software exp

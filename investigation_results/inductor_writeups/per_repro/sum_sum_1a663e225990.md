## Diagnosis

The captured graph builds a dense one-hot equality mask from labels, multiplies
it by the scalar loss scale, reduces that one-hot tensor over vocabulary, uses
the scalar row result in a dense `exp(logits - row_offsets)` softmax-backward
epilogue, adds the incoming base gradient, then returns both padded
materializations and a sibling column sum.

The oracle eliminates the one-hot tensor algebraically. For valid labels, the
dense update is `base + scale * exp(...)` everywhere with a `-scale` correction
at the single label column; invalid labels leave the base unchanged. This is an
`ALGEBRAIC_ELIMINATION` opportunity rather than a true pool/upsample
scatter-reduce. Inductor would need to canonicalize this one-hot-mask plus
row-reduction pattern and fuse the remaining multi-output stores with the
column reduction.

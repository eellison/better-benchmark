## Root cause

The channel reduction over dims [0, 2, 3] with only 16 output channels means the reduction numel is 512 * 55 * 55 = 1,548,800 per channel. Inductor's generic split-K does not exploit the NCHW spatial structure -- it could use the fixed 3025-element spatial tiles (one H*W plane) as natural first-stage units, giving exactly 512 independent partial sums per channel that can be accumulated more efficiently.

With only 16 output channels, the problem has low parallelism in the output dimension. The oracle uses NCHW-aware tiling that gives each CTA a contiguous spatial tile to reduce, avoiding the uneven chunk boundaries that Inductor's flattened reduction creates.

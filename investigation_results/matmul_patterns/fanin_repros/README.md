# Matmul fan-in repros

Standalone, runnable repros of the matmul **fan-in** pattern: ≥2 independent
matmuls whose outputs are combined — through a pointwise bridge — into ONE
output. Unlike the corpus canonical repros (which store only the bridge, since
matmuls are extern ops the partitioner cuts out), these glue the matmuls back
in, so the whole fan-in runs and `torch.compile`s as a single graph.

One directory per UNIQUE fan-in signature (deduped across the corpus). Each
`repro.py` runs eager + `torch.compile` from `__main__`:

```bash
python investigation_results/matmul_patterns/fanin_repros/fanin_2x_mm_mul_e32c620d72/repro.py
```

Naming: `fanin_<n>x_<matmul op>_<combine op>_<hash>`.

- `*_mul_*`  — gated / SwiGLU MLP: `gelu/silu(mm(x,Wg)) * mm(x,Wu)`
  (gemma, qwen, mistral, mt5). Both projections feed only the bridge →
  the intermediates never need to hit DRAM.
- `*_add_*`  — residual / parallel projection: `mm(a,b) + mm(c,d)`
  (vit/deit, encoder-decoder family; `12x` is Albert's weight-tied
  cross-layer accumulation).

Generated by `scripts/matmul_pattern_analysis.py --emit-fanin-repros`.
